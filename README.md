# FateDecoder

生年月日・出生地をもとに複数の占術を統合し、コミュニケーションの「らしさ」を可視化するWebアプリケーション。

## 機能

- **Short Reading / Full Reading**: タロットと既存占術を使った鑑定
- **相性診断**: 2名の生年月日を入力した相性チェック
- **12軸プロファイル（らしさ）**: 生年月日・出生地から12の軸でらしさを数値化

## 12軸プロファイル機能

### 概要

「コミュニケーション翻訳システム」の基盤となる機能。11種類の占術（西洋占星術・四柱推命・蔵干・Gene Keys・Human Design・易経・マヤ暦・数秘術・算命学・宿曜・九星気学）を統合し、12の軸でその人の傾向パターンを −10〜+10 のスコアで可視化する。

占術名・生データはUIに一切表示せず「複数の心理測定モデルと傾向データを統合した分析結果」として提示する（抵抗感対策）。

### 12軸

| 軸 | 左極 | 右極 |
|----|------|------|
| 判断 | 論理 | 感情 |
| エネルギー方向 | 内向 | 外向 |
| 情報の捉え方 | 直感 | 現実 |
| 行動様式 | 構造 | 流動 |
| 対人距離 | 協調 | 独立 |
| 社会性 | 個 | 群 |
| 変化への姿勢 | 保守 | 革新 |
| 生き方の重心 | 在り方 | やり方 |
| 自己表現 | 受容 | 主張 |
| 他者理解 | 共感 | 客観 |
| 感性 | 粗 | 繊細 |
| 創造性 | 再現 | 創造 |

### `/rashisa` ページの使い方

1. `/rashisa` にアクセス
2. お名前（ニックネーム可）を入力
3. 生年月日（年・月・日）を入力
4. 出生時間を選択（不明な場合は 12:00 のままで可）
5. 出生地を入力フィールドの一覧から選択（対応25都市）
6. 「12軸プロファイルを見る」を押す

**対応都市（25都市）**: 東京・大阪・名古屋・福岡・札幌・仙台・広島・神戸・京都・横浜・千葉・沖縄・那覇・鹿児島・熊本・長崎・新潟・金沢・ニューヨーク・ロンドン・パリ・北京・上海・ソウル・シンガポール

### `/api/rashisa` API 仕様

#### リクエスト

```
POST /api/rashisa
Content-Type: application/json
```

```json
{
  "name": "ニックネーム",
  "birthDate": "1989-10-24",
  "birthTime": "05:58",
  "latitude": 34.6937,
  "longitude": 135.5023,
  "timezone": "Asia/Tokyo"
}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|---|------|------|
| name | string | 必須 | 名前またはニックネーム |
| birthDate | string | 必須 | YYYY-MM-DD 形式 (1900〜2100) |
| birthTime | string | 必須 | HH:MM 形式 |
| latitude | number | 必須 | 緯度 (-90〜90) |
| longitude | number | 必須 | 経度 (-180〜180) |
| timezone | string | 任意 | タイムゾーン (例: "Asia/Tokyo") |

#### レスポンス（成功）

```json
{
  "profile": {
    "axes": [
      {
        "name": "判断",
        "polarity": ["論理", "感情"],
        "rashisaScore": 1.1
      }
    ]
  }
}
```

#### エラーレスポンス

| ステータス | 説明 |
|-----------|------|
| 400 | バリデーションエラー（必須フィールド不足・形式不正） |
| 403 | Origin 制限による拒否 |
| 429 | レート制限超過（Retry-After: 60 ヘッダー付き） |
| 500 | サーバー内部エラー |

### 注意事項

- **サーバーサイド専用**: sweph（天文計算ライブラリ）依存のためブラウザでは動作しない
- **Node ランタイム必須**: `export const runtime = 'nodejs'` が必要
- データは保存されない（ステートレス計算）

## 環境変数

| 変数名 | 必須 | 説明 |
|--------|------|------|
| `ALLOWED_ORIGINS` | 任意 | カンマ区切りの許可Origin（例: `https://v0-fate-decoder.vercel.app`）。未設定時は全許可（開発環境用） |
